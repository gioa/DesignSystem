How to choose

How to Choose

Disable vs. Hide

  • Default to hide: Disabled fields give extra thoughts for the user about why it is not available. If they do not need it anyway, we should hide it.

  • When to disable:

    • Part of a sentence: e.g., Auto-terminate after X hours of inactivity, the user can enable it by clicking the checkbox.
    • Discovery: e.g. you could have this feature if you are enterprise user.
    • Constantly hiding and showing fields may distract because the view changes constantly. E.g., EBS fields.

Load more vs. Pagination vs. Infinite scrolling

  • Load more: single unambiguous choice. Should allow to view all at once, or show no. of all results to avoid leaving your users guessing when they are going to run out of results.

  • Pagination: each page is a specific set of results, user knows item location and have a sense of control. But it is perceived to be slow, discourages users from browsing.

  • Infinite scrolling: more items appear as the user scroll down the page. It works well for discovery websites, but is often disastrous for prioritization apps.
